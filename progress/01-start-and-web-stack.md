## Создание проекта FlightSim

Для практики я начала строить собственный production-подобный веб-сервис.

Проект находится в каталоге:


`/opt/flightsim`

В проекте используются:

- Ubuntu;
- nginx;
- Gunicorn;
- Flask;
- Python virtual environment;
- systemd.

-- Что я поняла про компоненты--

Я разобралась в их основном назначении:

- nginx принимает внешние HTTP-запросы;
- nginx работает как reverse proxy;
- nginx передаёт запросы локальному Gunicorn;
- Gunicorn запускает Python-приложение;
- Flask содержит маршруты и логику приложения;
- nginx не читает app.py и не выполняет Python-код;
- Gunicorn слушает 127.0.0.1:8000

Я также разобрала различия между:

localhost;
127.0.0.1;
внешним IP адресом;
локальным портом;
сервисом, доступным из интернета.


## Gunicorn и systemd

Я познакомилась с двумя способами запуска Gunicorn:
ручной запуск из терминала;
автоматический запуск через systemd
Gunicorn запускается через сервис:

- `flightsim.service`

Я поняла, что ручной Gunicorn и Gunicorn, запущенный через systemd, являются отдельными процессами.

Если оба процесса пытаются слушать один порт, может появиться ошибка:

`Address already in use`

Эта ошибка означает, что указанный адрес и порт уже заняты другим процессом.
