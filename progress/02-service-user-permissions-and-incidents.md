
# Часть 2. Service user, права Linux и первые инциденты

## Проблема запуска от root

Первоначально приложение работало с избыточными привилегиями.

Я разобралась, почему веб-приложение не должно запускаться от `root`:

- `root` имеет полный контроль над системой;
- приложение работает с правами своего процесса;
- при компрометации приложения атакующий может получить права этого процесса;
- отдельный ограниченный пользователь уменьшает возможный ущерб;
- приложению не нужны права на всю операционную систему.

## Создание service user

Я создала отдельного системного пользователя для FlightSim:

```text
mrpopo
```
Его параметры:
```text
shell: /usr/sbin/nologin
home: /opt/flightsim
group: mrpopo
sudo: отсутствует
```
