# Часть 3. Flask templates и знакомство с Jinja2

## Исходное состояние

Первоначально Flask возвращал HTML непосредственно из Python-функций.

Это смешивало:

- Python-логику;
- маршруты;
- HTML-разметку.

На сайте уже существовали маршруты:

- `/`;
- `/news`;
- `/pilots`;
- `/status`.

## Перенос HTML в шаблон

Я создала директорию:

`/opt/flightsim/templates`

В app.py был импортирован:

`from flask import Flask, render_template`

Маршрут / начал возвращать:

`return render_template("index.html")`

Что я поняла

Я разобралась в разделении ответственности:

- app.py отвечает за Python-логику и маршруты;
- templates/index.html содержит HTML;
- Flask ищет шаблоны в директории templates;
- render_template() загружает шаблон;
- результатом работы Flask остаётся готовый HTML-ответ.

После изменений я проверяла:

- синтаксис Python через python -m py_compile;
- Gunicorn напрямую через curl;
- сайт через nginx;
- журнал сервиса через journalctl.
- Создание общего шаблона

- ссылки входа и регистрации;
- левое меню;
- общая область содержимого.

Главная страница осталась в:

`templates/index.html`

## Знакомство с Jinja2

Я начала разбирать конструкции:

- {% extends "base.html" %}
- {% block title %}
- {% block content %}
- {% endblock %}

Я поняла основную идею наследования:

- base.html является общим каркасом;
- index.html наследует этот каркас;
- block title позволяет задать уникальный заголовок страницы;
- block content обозначает место для уникального содержимого;
- повторяющийся HTML не нужно копировать во все шаблоны.

Я пока продолжаю изучать Jinja2. Общую схему наследования я уже понимаю, но внутренний процесс обработки шаблонов Flask ещё разбираю.
